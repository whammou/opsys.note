* WebDAV Service Documentation
:PROPERTIES:
:CUSTOM_ID: webdav-service-documentation
:END:
** Overview
:PROPERTIES:
:CUSTOM_ID: overview
:END:
This document describes the systemd user service for running a WebDAV
server using rclone, with a focus on secure secret management and
security hardening.

** Service File
:PROPERTIES:
:CUSTOM_ID: service-file
:END:
*Location*: =~/.config/systemd/user/webdav@.service=

#+begin_src ini
[Unit]
Description=journal webdav client
After=network.target

[Service]
LoadCredential=webdav-htpasswd:%h/.config/rclone/webdav.htpasswd
ExecStart=/usr/bin/rclone serve webdav \
    --dir-cache-time=1s \
    --poll-interval=1s \
    --htpasswd=%d/webdav-htpasswd \
    --addr=0.0.0.0:34553 \
    %h/%i/
#Restart=on-failure

# Security hardening
ProtectSystem=strict
ProtectHome=hidden
PrivateTmp=yes
NoNewPrivileges=true

[Install]
WantedBy=default.target
#+end_src

** Security Features
:PROPERTIES:
:CUSTOM_ID: security-features
:END:
*** 1. systemd Credentials (LoadCredential)
:PROPERTIES:
:CUSTOM_ID: systemd-credentials-loadcredential
:END:
#+begin_src ini
LoadCredential=webdav-htpasswd:%h/.config/rclone/webdav.htpasswd
#+end_src

- Credentials are mounted as read-only files in =$CREDENTIALS_DIRECTORY=
- Location: =/run/user/<uid>/credentials/webdav@Journal.service/=
- Accessed via =%d/webdav-htpasswd= in ExecStart
- NOT visible in process list (=ps aux=)

*Benefits*: - Secrets never appear in =/proc/<pid>/cmdline= - Automatic
cleanup when service stops - Supports encryption with
=systemd-creds encrypt=

*** 2. Security Hardening
:PROPERTIES:
:CUSTOM_ID: security-hardening
:END:
| Directive              | Purpose                                              |
|------------------------+------------------------------------------------------|
| =ProtectSystem=strict= | Read-only filesystem (except =/usr=, =/etc=, =/run=) |
| =ProtectHome=hidden=   | Home directory completely inaccessible               |
| =PrivateTmp=yes=       | Isolated =/tmp= namespace                            |
| =NoNewPrivileges=true= | Prevent privilege escalation                         |

*** 3. htpasswd Authentication
:PROPERTIES:
:CUSTOM_ID: htpasswd-authentication
:END:
Instead of passing plaintext password via CLI:

#+begin_src ini
# AVOID - password visible in process list
ExecStart=... --user=foo --pass=secret123
#+end_src

We use htpasswd with bcrypt:

#+begin_src ini
# SECURE - password hash in separate file
ExecStart=... --htpasswd=%d/webdav-htpasswd
#+end_src

*Creating htpasswd file*:

#+begin_src sh
# Create with bcrypt (recommended)
htpasswd -Bc ~/.config/rclone/webdav.htpasswd username

# Set secure permissions
chmod 600 ~/.config/rclone/webdav.htpasswd
#+end_src

** Systemd Specifiers Used
:PROPERTIES:
:CUSTOM_ID: systemd-specifiers-used
:END:
| Specifier | Meaning               | Example                                             |
|-----------+-----------------------+-----------------------------------------------------|
| =%u=      | User name             | =whammou=                                           |
| =%h=      | User's home directory | =/home/whammou=                                     |
| =%i=      | Instance name         | =Journal=                                           |
| =%n=      | Full instance name    | =webdav@Journal=                                    |
| =%N=      | Instance name         | =Journal=                                           |
| =%d=      | Credential directory  | =/run/user/1000/credentials/webdav@Journal.service= |

** Managing the Service
:PROPERTIES:
:CUSTOM_ID: managing-the-service
:END:
#+begin_src sh
# Start service
systemctl --user start webdav@Journal

# Stop service
systemctl --user stop webdav@Journal

# Restart service
systemctl --user restart webdav@Journal

# Check status
systemctl --user status webdav@Journal

# View logs
journalctl --user -u webdav@Journal -f

# Enable at boot
systemctl --user enable webdav@Journal

# Reload after editing
systemctl --user daemon-reload
#+end_src

** Advanced: Encrypted Credentials
:PROPERTIES:
:CUSTOM_ID: advanced-encrypted-credentials
:END:
For additional security, encrypt the credential file:

#+begin_src sh
# Encrypt credential
systemd-creds encrypt --name=webdav-htpasswd \
    ~/.config/rclone/webdav.htpasswd \
    ~/.config/rclone/webdav.htpasswd.encrypted

# Update service
LoadCredentialEncrypted=webdav-htpasswd:%h/.config/rclone/webdav.htpasswd.encrypted
#+end_src

Note: Encrypted credentials require =systemd-creds decrypt= at boot or a
TPM2 chip.

** Comparison: Secret Management Methods
:PROPERTIES:
:CUSTOM_ID: comparison-secret-management-methods
:END:
| Method                  | Secret in ps? | Encrypted at Rest | systemd-native | Notes                   |
|-------------------------+---------------+-------------------+----------------+-------------------------|
| =bash -c $VAR=          | ✓             | ✗                 | ✗              | Not recommended         |
| =--pass=...=            | ✓             | ✗                 | ✓              | Basic                   |
| htpasswd                | ✗             | ✓                 | ✓              | *Recommended*           |
| LoadCredential          | ✗             | Optional          | ✓              | Best for files          |
| LoadCredentialEncrypted | ✗             | ✓                 | ✓              | Highest security        |
| pass (GPG)              | ✗             | ✓                 | ✗              | Requires key management |

** Troubleshooting
:PROPERTIES:
:CUSTOM_ID: troubleshooting
:END:
*** Service fails to start
:PROPERTIES:
:CUSTOM_ID: service-fails-to-start
:END:
#+begin_src sh
# Check detailed logs
journalctl --user -u webdav@Journal -xe

# Verify credential file exists and is readable
cat ~/.config/rclone/webdav.htpasswd
#+end_src

*** Authentication fails
:PROPERTIES:
:CUSTOM_ID: authentication-fails
:END:
#+begin_src sh
# Regenerate htpasswd
htpasswd -Bc ~/.config/rclone/webdav.htpasswd username
# Enter password when prompted

# Verify permissions
ls -la ~/.config/rclone/webdav.htpasswd
# Should be: -rw-------
#+end_src
