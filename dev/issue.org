#+name: Description
#+begin_quote org
Usually when behind a public or locked down firewall majority of the outbound traffic allowed id port 80(http) and port 443(https). Due to mostly security concerns. With this reverse proxy setup we can mask all of our traffic to appear as essentially HTTPS and traffic which allows us to bypass a lot of these restrictions
#+end_quote

#+name: Setup and Bypass Sequence
#+begin_src mermaid

    Note over VPS: Phase 1: Server Setup
    VPS->>VPS: Install Apache2 + open ports 22,80,443
